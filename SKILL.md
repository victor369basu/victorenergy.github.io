## Phase 1: Understand Before Building

Extract these before writing a single line of code:

1. **Purpose & conversion goal** — What is the ONE action the site must drive? (Buy, Sign up, Book, Contact?)
2. **Audience & persona** — Who lands here? What motivates them? What do they fear?
3. **Brand tone** — Serious/corporate? Aggressive/athletic? Playful? Luxury? Technical?
4. **Reference sites** — If the user names sites, fetch and study them (`web_fetch`). Extract: nav structure, hero layout, section order, color mood, typography weight, motion feel, product showcase approach.
5. **Content inventory** — Products/services, flavors/variants, pricing, offers, testimonials, ingredients, USPs.
6. **Color & visual preferences** — If none given, derive from brand tone + reference sites.

**If references are given, always `web_fetch` them first.** Study the actual markup patterns, not just the screenshots.

## Phase 2: Design Direction

Commit to ONE clear aesthetic direction. Do not mix styles.

| Direction | Feel | Font Pairing | Best For |
|---|---|---|---|
| Bold/Athletic | Dark, high-contrast, cinematic | Bebas Neue + Barlow Condensed | Sports, energy, fitness products |
| Luxury/Editorial | Generous whitespace, serif-led | Playfair Display + DM Sans | Fashion, premium goods |
| Bold/Startup | Energetic, geometric, strong contrast | Cabinet Grotesk + Inter | SaaS, tech, apps |
| Organic/Natural | Warm, earthy, rounded | Fraunces + Nunito | Food, wellness, sustainability |
| Technical/SaaS | Clean, structured, monospaced | Syne + JetBrains Mono | Developer tools, APIs |
| Brutalist | Raw, oversized type, minimal color | Space Grotesk + Courier | Portfolio, art, fashion-forward |

**Victor project used:** Bold/Athletic — Bebas Neue (display/headings) + Barlow Condensed (labels/CTAs) + Barlow (body). Pure black backgrounds, electric yellow (#E9FF57) accent, flavor-specific color themes.

**Anti-patterns — never do these:**
- Hero with centered H1 + subtitle + two buttons on white/light bg (boring)
- Purple-to-blue gradient on white (overused)
- Inter font + blue CTA (default AI output look)
- Generic card grid with box-shadow on light gray bg (cliché)
- Lorem ipsum or placeholder text anywhere
- "Feature 1, Feature 2" non-specific copy

## Phase 3: Site Architecture

A complete product/e-commerce site structure (adapt to context):

```
1.  Navigation           — Fixed, transparent → frosted glass on scroll, dropdown menus, cart icon
2.  Hero                 — Full-bleed, cinematic, product-centered, 3-column layout (text | product | stats)
3.  FOMO Bar             — Live buyer count, countdown timer, review rating — drives urgency
4.  Proof Ticker         — Scrolling marquee of brand slogans or stats in accent color
5.  Media / "As Seen In" — Scrolling logo/name strip to build credibility instantly
6.  Product Showcase     — Per-product full-bleed panels with sticky tab navigation (PRIME-style)
7.  Product Grid         — Pepsi-style image-led product cards with hover animations
8.  Formula / Features   — Ingredient or feature deep-dive, numbered cards
9.  Offers / Pricing     — Discount cards with countdown timers, clear CTAs
10. Social Wall          — Instagram-style user content grid ("Tag us @brand")
11. Reviews              — Dark grid testimonials with star ratings and real-sounding names
12. Final CTA            — Full-bleed, centered, glow effect, single focused action
13. Footer               — Brand logo, 3-4 link columns, socials, copyright, legal links
14. Sticky Add-to-Cart   — Fixed bottom bar visible after hero scrolls out of view
15. Checkout Modal       — Multi-step (flavor pick → delivery → payment → success)
```

Not every site needs all sections. Minimal services site: Nav + Hero + Social proof + Features + Testimonials + CTA + Footer.

For light-mode sites, invert: `--blk:#fff; --dk:#f5f5f5; --txt:#111;`

### 4.2 Typography Scale

Use `clamp()` everywhere for fluid scaling:

```css
/* Display */     font-size: clamp(5rem, 15vw, 14rem);
/* H1 */          font-size: clamp(4rem, 8vw, 8rem);
/* H2 */          font-size: clamp(3rem, 6vw, 6rem);
/* H3 */          font-size: clamp(1.8rem, 3vw, 2.4rem);
/* Body */        font-size: 1rem;
/* Small */       font-size: 0.875rem;
/* Label */       font-size: 0.7rem; letter-spacing: 0.18em; text-transform: uppercase;
```

Heading line-height: 0.88–0.95. Body line-height: 1.7–1.85.

### 4.3 Spacing Scale

Always use `clamp()` for padding on sections:

```css
.sec { padding: clamp(5rem, 10vw, 9rem) clamp(1.5rem, 5vw, 5rem); }
```

### 5.2 Scroll Reveal System

**CRITICAL RULE:** Never apply `.rv` or `.reveal` class to elements inside `display:none` containers. IntersectionObserver cannot fire for hidden elements — they stay invisible forever. Always force `opacity:1` on content inside conditionally shown panels.

```javascript
const ro = new IntersectionObserver(entries => {
  entries.forEach(e => {
    if (e.isIntersecting) { e.target.classList.add('in'); ro.unobserve(e.target); }
  });
}, { threshold: 0.1 });
document.querySelectorAll('.rv, .rg').forEach(el => ro.observe(el));
```

```css
.rv { opacity:0; transform:translateY(36px); transition:opacity .75s var(--ease), transform .75s var(--ease); }
.rv.in { opacity:1; transform:none; }

/* Staggered group */
.rg > * { opacity:0; transform:translateY(22px); transition:opacity .65s var(--ease), transform .65s var(--ease); }
.rg.in > * { opacity:1; transform:none; }
.rg.in > *:nth-child(1) { transition-delay:0s }
.rg.in > *:nth-child(2) { transition-delay:.08s }
.rg.in > *:nth-child(3) { transition-delay:.16s }
.rg.in > *:nth-child(4) { transition-delay:.24s }
```

### 5.4 3D Product Tilt (mouse-tracked)

Apply to hero product images or cans. Creates an immersive premium feel:

```javascript
const wrap = document.getElementById('productWrapper');
document.addEventListener('mousemove', e => {
  const cx = window.innerWidth / 2, cy = window.innerHeight / 2;
  const dx = (e.clientX - cx) / cx, dy = (e.clientY - cy) / cy;
  wrap.style.transform = `rotateY(${dx * 18}deg) rotateX(${-dy * 12}deg) translateY(${-Math.abs(dy) * 8}px)`;
  wrap.style.transition = 'transform .1s linear';
});
wrap.addEventListener('mouseleave', () => {
  wrap.style.transform = '';
  wrap.style.transition = 'transform .6s var(--ease)';
});
```

Requires `transform-style: preserve-3d; perspective: 800px;` on the wrapper.

### 5.5 Particle Explosion (CTA click effect)

Fires 20 particles from button click position — color matches current active flavor/theme:

```javascript
function explodeParticles(btn, color = '#E9FF57') {
  const rect = btn.getBoundingClientRect();
  const cx = rect.left + rect.width/2, cy = rect.top + rect.height/2;
  for (let i = 0; i < 20; i++) {
    const p = document.createElement('div');
    p.className = 'px-particle';
    const ang = Math.random() * Math.PI * 2;
    const dist = 80 + Math.random() * 120;
    const sz = 4 + Math.random() * 8;
    p.style.cssText = `position:fixed;border-radius:50%;pointer-events:none;z-index:9999;
      left:${cx}px;top:${cy}px;width:${sz}px;height:${sz}px;background:${color};
      --pxX:${Math.cos(ang)*dist}px;--pxY:${Math.sin(ang)*dist}px;
      animation:pxFly ${.5+Math.random()*.4}s var(--ease) ${Math.random()*.1}s forwards`;
    document.body.appendChild(p);
    setTimeout(() => p.remove(), 900);
  }
}
```

### 5.7 Product Panel Switching (tab-based, like drinkprime.uk)

For multi-variant products. Each panel is a full-bleed section with its own color theme:

```javascript
function showPanel(key, btn) {
  // Hide all
  document.querySelectorAll('.flavor-panel').forEach(p => p.classList.remove('active'));
  document.querySelectorAll('.panel-tab').forEach(b => b.classList.remove('active'));

  const panel = document.getElementById('panel-' + key);
  panel.style.opacity = '0';
  panel.classList.add('active'); // display:block

  // Fade in panel
  requestAnimationFrame(() => {
    panel.style.transition = 'opacity .4s ease';
    panel.style.opacity = '1';
    // Animate product element inside
    const product = panel.querySelector('.product-wrap');
    if (product) {
      product.style.cssText = 'transform:scale(.82) translateY(40px);opacity:0;transition:transform .55s cubic-bezier(.16,1,.3,1),opacity .45s ease';
      requestAnimationFrame(() => { product.style.transform = ''; product.style.opacity = '1'; });
    }
  });
  if (btn) btn.classList.add('active');
}
```

**CRITICAL:** Panels that are `display:none` before activation must NOT have `.rv` reveal classes on their children — the IntersectionObserver can't fire for hidden elements. Force `opacity:1` with CSS override on panel children.

## Phase 6: SVG Product Illustrations

For products without photos, build photorealistic SVG illustrations. Key technique from Victor project:

### 6.1 Realistic Metallic Can

A believable aluminum can requires these layers in order:
1. **Body gradient** — dark left edge → lighter mid → bright highlight → back to dark (simulates cylindrical reflection)
2. **Vertical ridge lines** — subtle, `opacity:.04`, simulate manufacturing texture
3. **Label band** — darker rect with border lines top/bottom
4. **Brand text** — rotated ~8deg like real can labels (PRIME reference)
5. **Shine overlay** — soft white gradient left→center→right
6. **Edge darkness** — black gradient on left/right edges for 3D depth
7. **Neck taper** — `<path>` element narrowing toward the top
8. **Seam ring** — ellipse at shoulder
9. **Aluminum dome top** — ellipse with metallic gradient
10. **Pull tab + ring** — ellipses + rect + circle rivet

**CRITICAL: SVG defs ID uniqueness.** When rendering the same can design multiple times on one page (hero, product card, flavor panel), each instance MUST use unique gradient/filter IDs. Duplicate IDs cause browsers to reuse the first definition, corrupting all others.

```javascript
// Always pass a unique prefix per render location
function buildCan(config, uniquePrefix) {
  const uid = uniquePrefix; // e.g. 'hero', 'fp_strawberry', 'pc_coconut'
  return `<defs>
    <linearGradient id="${uid}Body" ...>...</linearGradient>
    <filter id="${uid}Glow">...</filter>
    <clipPath id="${uid}Clip">...</clipPath>
  </defs>
  <rect ... fill="url(#${uid}Body)" clip-path="url(#${uid}Clip)"/>
  ...`;
}

// Usage — different uid per location
document.getElementById('heroCan').innerHTML = buildCan(cfg, 'hero');
document.getElementById('can-strawberry').innerHTML = buildCan(cfg, 'fp_strawberry');
document.getElementById('pcan-strawberry').innerHTML = buildCan(cfg, 'pc_strawberry');
```

```javascript
(function() {
  const end = Date.now() + 2 * 60 * 60 * 1000; // 2h from page load
  function tick() {
    const rem = Math.max(0, end - Date.now());
    const h = Math.floor(rem / 3600000);
```

### 7.8 Multi-Step Checkout Modal

Three steps: Product selection → Delivery + Payment → Success.

**Step 1:** Flavor/variant picker, quantity control, offer code pills, live price breakdown.

**Step 2:** Delivery form (name, email, phone, address, city, pin), payment method selector (UPI / Card / COD), order summary pill.

**Step 3:** Success state with animation, savings summary, delivery timeframe.

**Offer code logic:**
```javascript
function calculateTotal(qty, offerCode) {
  const base = PRICE_PER_UNIT * qty;
  const delivery = 49;
  let discount = 0;
  if (offerCode === 'LAUNCH20') discount = Math.round(base * .2);
  if (offerCode === 'COMBO3' && qty >= 2) discount = PRICE_PER_UNIT; // one free
  if (offerCode === 'BULK30' && base >= 2000) discount = Math.round(base * .3);
  if (offerCode === 'STUDENT10') discount = Math.round(base * .1);
  return { base, discount, total: base - discount + delivery };
}
```

**UPI ID and Card formatting:**
```javascript
function formatCard(input) {
```

## Phase 11: Performance & Code Quality

- **Single file** — all CSS in `<style>`, all JS in `<script>` before `</body>`. No external JS dependencies.
- **Google Fonts** — one `<link>` tag, only load weights actually used.
- **CSS variables** — all colors, radii, and easing via `:root` vars. No magic numbers in component CSS.
- **No `!important` except** for critical override situations (e.g., forcing `.rv` opacity inside hidden panels).
- **SVG inline** — never use `<img src="...">` for icons; inline SVG paths.
- **Avoid `document.write`** — always `innerHTML`, `textContent`, or DOM API.
- **Event delegation** — use `onclick` attributes for simple interactions (reduces boilerplate for single-page sites).
- **Intersection Observer** — always `unobserve` after triggering to save memory.
- **Particle cleanup** — always `setTimeout(() => el.remove(), duration)` on dynamically created particles.

## Phase 12: Common Mistakes to Avoid

These issues caused bugs in the Victor project — avoid in all future projects:

| Mistake | Symptom | Fix |
|---|---|---|
| `.rv` class on children of `display:none` panel | Content stays invisible after panel is shown | Remove `.rv` from panel children; add `opacity:1!important` override in CSS |
| Duplicate SVG `<defs>` IDs across multiple renders | Wrong colors/gradients on subsequent cans | Always pass unique `uid` prefix to buildCan() per render location |
| SVG `<text>` without `overflow:hidden` on container | Text leaks outside SVG bounds, appears in wrong page area | Set `overflow:hidden` on SVG wrapper or use `<clipPath>` |
| `display:none` panel with `display:flex` when active | Panel collapses due to specificity conflict | Use `display:block` for panel, set inner to flex; or use `!important` |
| Reveal animations blocking panel content | Elements never become visible inside toggled panels | Use JS to add `.in` class after panel is shown, not IntersectionObserver |
| Same gradient IDs across multiple SVG elements | Browser reuses first definition, all others wrong color | Unique ID per instance — use prefix pattern |
| `animation` CSS fighting JS `opacity` transitions | Can appears/disappears erratically | Wrap product in `.product-wrap` div; animate div separately from SVG |

## Phase 13: Content & Copy Guidelines

- **Never** use lorem ipsum, "Feature 1", "Your company name", or generic placeholder text.
- **Stats** should be specific and believable: "50,000+ customers", "247 bought today", "4.9/5 stars"
- **Testimonials** need: real-sounding name, specific role + city, a concrete story, not generic praise
- **Ingredient/feature descriptions** must explain the mechanism: not "Caffeine boosts energy" but "150mg precision-dosed caffeine — activates alertness in 15 minutes, no spike-and-crash cycle"
- **CTA copy** should create tension or urgency: not "Buy Now" but "Add to Cart ⚡" or "Fuel Up Now"
- **Eyebrow labels** above headings in small caps add editorial quality: `⚡ New Launch — Limited Stock`
- **Offer codes** should be clever mnemonics: `LAUNCH20`, `COMBO3`, `BULK30`, `STUDENT10`

## Phase 15: Real Social & External Link Wiring

When a user provides real social media profile URLs, wire every related link across the entire site — don't just update one spot. Placeholder `href="#"` links must never ship on a real site.

### 15.1 Collect URLs First

Before writing any link, confirm which platforms the user has. Common mapping:

| Platform | Icon Used | Typical Use |
|---|---|---|
| Instagram | 📸 | Brand lifestyle, product shots, community |
| YouTube | ▶ | Product videos, ads, tutorials |
| X / Twitter | 𝕏 | Announcements, engagement |
| Facebook | f | Community, ads, events |
| LinkedIn | in | Founder profile, B2B, press |
| TikTok | ♪ | Short-form video, gen-Z reach |

If a user only has 2–3 platforms, **map multiple icons to the same URL** rather than leaving dead `#` links. For example, if they have Instagram + LinkedIn but no YouTube:
- 📸 Instagram → their Instagram URL
- ▶ YouTube → their Instagram URL (best available alternative)
- 𝕏 X → their LinkedIn URL
- f Facebook → their LinkedIn URL

### 15.2 All Link Locations to Update

Never update just one area. Scan and fix every occurrence:

```
1. Footer social icon buttons      (.fsoc / .footer-social links)
2. Footer column links             (Contact, Careers, Press, About — replace # with real URL)
3. Mobile menu links               (if social links appear in mobile nav)
4. Hero CTA secondary links        (if "Follow Us" or similar)
5. Instagram/Social wall section   (each card onclick should open the social profile)
6. Any inline "Follow @handle" text links in the body
```

### 15.3 Link Best Practices

Always use these attributes on external links:

```html
<a
  href="https://www.instagram.com/yourbrand/"
  target="_blank"
  rel="noopener noreferrer"
  title="Instagram"
>
  📸
</a>
```

- `target="_blank"` — opens in new tab (user doesn't lose the site)
- `rel="noopener noreferrer"` — security: prevents the new tab from accessing `window.opener`; also prevents referrer leakage
- `title="..."` — tooltip on hover, improves accessibility

### 15.4 Footer Column Link Replacement Pattern

When the user has limited social URLs, replace generic footer links with the most relevant real destination:

```html
<!-- BEFORE (dead links) -->
<div class="fcol">
  <h4>Company</h4>
  <ul>
    <li><a href="#">About Us</a></li>
    <li><a href="#">Careers</a></li>   <!-- dead -->
    <li><a href="#">Press</a></li>     <!-- dead -->
  </ul>
</div>
<div class="fcol">
  <h4>Help</h4>
  <ul>
    <li><a href="#">Contact</a></li>   <!-- dead -->
  </ul>
</div>

<!-- AFTER — real Instagram used for Contact/Careers/Press -->
<div class="fcol">
  <h4>Company</h4>
  <ul>
    <li><a href="#">About Us</a></li>
    <li><a href="https://www.instagram.com/yourbrand/" target="_blank" rel="noopener noreferrer">Careers</a></li>
    <li><a href="https://www.instagram.com/yourbrand/" target="_blank" rel="noopener noreferrer">Press</a></li>
  </ul>
</div>
<div class="fcol">
  <h4>Help</h4>
  <ul>
    <li><a href="https://www.instagram.com/yourbrand/" target="_blank" rel="noopener noreferrer">Contact</a></li>
  </ul>
</div>
```

**Rule:** If no dedicated page exists for a link, route it to the most appropriate social/contact channel rather than leaving `href="#"`. Instagram DMs are a valid contact method for small brands.

### 15.5 Social Wall Click Targets

The Instagram-style social wall grid should link each card to the real Instagram profile:

```html

<div class="ig-card" onclick="window.open('https://www.instagram.com/yourbrand/','_blank')">
```

Or use an `<a>` wrapper instead of onclick for better accessibility:

```html

  <div class="ig-inner">🍓⚡</div>
  <div class="ig-overlay"><span>View on Instagram</span></div>
</a>
```

After wiring all links, run this mental audit:

```

✓ Every social icon has a real URL
✓ All external links have target="_blank" rel="noopener noreferrer"
```
